# 로또 자동 구매 사용법

## 1. 이 프로젝트가 하는 일

이 프로젝트는 GitHub Actions를 이용해 다음 작업을 자동으로 처리합니다.

- 매주 월요일 동행복권 로또 6/45를 자동 구매하고, 설정값에 따라 연금복권 720+도 추가 구매
- 토요일에 로또 6/45와 연금복권 720+ 당첨 여부 확인
- 구매 완료 및 당첨 결과를 카카오톡으로 알림
- 구매한 번호를 `data/purchased.json`에 저장
- 연금복권 구매 내역을 `data/pension_purchased.json`에 저장

## 2. 준비물

실행 전에 아래 정보가 필요합니다.

- 동행복권 아이디
- 동행복권 비밀번호
- Gemini API 키
- 카카오 REST API 키
- 카카오 리프레시 토큰

또한 동행복권 예치금이 `구매 장수 × 1,000원` 이상 있어야 합니다.

## 3. 환경 변수 설정

GitHub Secrets에 아래 값을 등록합니다.

| Secret                | 설명                 |
| --------------------- | -------------------- |
| `LOTTO_ID`            | 동행복권 아이디      |
| `LOTTO_PW`            | 동행복권 비밀번호    |
| `API_KEY`             | Gemini API 키        |
| `KAKAO_REST_API_KEY`  | 카카오 REST API 키   |
| `KAKAO_REFRESH_TOKEN` | 카카오 리프레시 토큰 |

로컬 실행은 기본적으로 `DRY_RUN=true`라서 테스트 모드입니다.
실제 구매나 실제 확인이 필요하면 `DRY_RUN=false`를 명시해서 실행합니다.

## 4. 구매 장수 설정

구매 장수는 루트의 `settings.json`에서 조정합니다.

```json
{
  "total_tickets": 3,
  "pension_tickets": 0
}
```

`total_tickets` 값이 3이면 로또 3장을 구매하고, `pension_tickets` 값이 0이 아니면 그 수만큼 연금복권을 추가로 구매합니다.

## 5. 실행 방법

이 프로젝트는 GitHub Actions에서 실행하는 방식이 기본입니다.

GitHub 저장소에서:

1. `Actions` 탭으로 이동
2. `🎰 로또 자동 구매` 워크플로 선택
3. `Run workflow` 클릭

## 6. 실행 모드

워크플로 실행 시 아래 모드를 선택할 수 있습니다.

| 모드        | 설명                   |
| ----------- | ---------------------- |
| `buy`       | 구매만 실행            |
| `check`     | 당첨 확인만 실행       |
| `buy+check` | 구매 후 바로 당첨 확인 |

`buy+check`는 테스트용으로 사용하면 좋습니다.

## 7. 당첨 확인에 사용할 번호 직접 넣기

`check` 모드에서 구매 내역 파일 대신 직접 번호를 넣고 싶다면 `numbers` 항목에 아래 형식으로 입력합니다.

```text
[[4,8,15,23,31,38],[7,13,20,29,35,42]]
```

이 형식은 "번호 6개짜리 배열들의 목록"입니다.

## 8. 자동 저장 파일

구매가 완료되면 다음 파일에 번호가 저장됩니다.

- `data/purchased.json`
- `data/pension_purchased.json`

이 파일은 구매 내역 확인과 당첨 확인에 사용됩니다.

테스트 모드에서는 아래 파일에 따로 저장됩니다.

- `data/purchased.test.json`
- `data/pension_purchased.test.json`

## 9. 번호 선택 방식

현재 번호 선택은 Gemini AI 추천을 우선으로 사용합니다.

- `API_KEY`가 있으면 Gemini가 번호를 추천합니다.
- `API_KEY`가 없거나 API 오류가 나면 랜덤 번호로 대체합니다.

## 10. 주의할 점

- 예치금이 부족하면 구매가 중단됩니다.
- 카카오 관련 키가 없으면 알림은 건너뜁니다.
- 구매 실패나 조회 실패가 나면 GitHub Actions 로그를 확인해야 합니다.
- `DRY_RUN=true`로 실행하면 구매 버튼을 누르지 않고 선택 흐름만 테스트할 수 있습니다.
- GitHub Actions 워크플로우는 `DRY_RUN=false`로 설정되어 실제 실행 기준입니다.

## 11. 파일 구조 참고

주요 파일은 아래와 같습니다.

```text
src/lotto_buyer.py      # 자동 구매
src/check_number.py     # 당첨 확인
src/number_selector.py  # 번호 선택
src/kakao_notify.py     # 카카오톡 알림
settings.json           # 구매 장수 설정
data/purchased.json     # 구매 번호 저장
data/pension_purchased.json # 연금복권 구매 내역 저장
data/purchased.test.json # 로또 테스트 저장
data/pension_purchased.test.json # 연금복권 테스트 저장
```
